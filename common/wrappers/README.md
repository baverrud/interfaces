real-world AXI configurations used by Xilinx Zynq devices.  These were
cross-referenced against our `axi3`, `axi4`, and `axilite` interface
definitions in `../`.

## Known incompatibility

Vivado auto-generated wrappers use `STD_LOGIC_VECTOR(0 to 0)` for
single-bit handshakes (`arvalid`, `awready`, etc.).  Our SV interfaces
use scalar `logic`.  A thin adapter is needed:

```systemverilog
assign if_sv.arvalid  = wrapper_port[0];
assign wrapper_port[0] = if_sv.arready;
```

This is a Vivado generation quirk, not an AXI spec violation тАФ our
interfaces are spec-correct.
