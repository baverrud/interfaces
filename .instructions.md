# FPGA Interface Library — Tool Run Instructions

## EDA Tool Initialization

Before running simulation or synthesis, initialize the toolchain using batch files in `c:\cmdtools\`. Each script updates `PATH` and preserves the original in `ORIG_PATH`.

| Tool | Command | Toolchain Location |
|---|---|---|
| **Questa 2025.2** | `c:\cmdtools\q25.bat` | `C:\altera\25.1std\questa_fse\win64` |
| **ModelSim 2020.1** | `c:\cmdtools\m20.bat` | `C:\intelFPGA\20.1\modelsim_ase\win32aloem` |
| **Vivado 2025.2** | `c:\cmdtools\v25.bat` | `C:\Xilinx\2025.2\Vivado` |
| **Vivado 2023.2** | `c:\cmdtools\v23.bat` | `C:\Xilinx\Vivado\2023.2` |

After sourcing, `vsim` / `vivado` are available for the rest of the shell session:

```batch
:: Simulation
c:\cmdtools\m20.bat
cd axi4\sv
python ../../common/scripts/engine.py sim . top_tb

:: Synthesis
c:\cmdtools\v23.bat
cd axi4\sv
python ../../common/scripts/engine.py synth . top
```

**Note:** `v23.bat` and `v25.bat` both run `vivado -version` as their last line. This causes a redundant Vivado invocation before the actual synthesis script runs — harmless, but adds a few seconds of startup delay.

### Expected synthesis warnings (benign)

| Warning | Cause |
|---|---|
| `Synth 8-7137` — "both Set and reset" | Standard `always_ff @(posedge clk or negedge rst)` pattern |
| `Synth 8-7129` — "port unconnected/no load" | AXI sideband signals tied to constant 0 (awsize, awburst, awcache, awprot, etc.) — optimized away |
| `Synth 8-6014` — "unused sequential element removed" | `wcnt` register eliminated during logic optimization |
| `Synth 8-4767` — "RAM dissolved into registers" | Register file has async reset, preventing block RAM inference |

## Vivado Synthesis

1. All synthesis runs happen in the `viv/` folder of the language variant.  
2. Never run from `rtl/`.  
3. Batch synthesis (no GUI):

   ```powershell
   cd examples/axistream/sv
   python ../../common/scripts/engine.py synth . top
   ```

4. For a targeted one-off synthesis of a specific top module (e.g. A/B proof):

   ```powershell
   cd examples/axistream/sv
   python ../../common/scripts/engine.py synth . top
   ```

## Source Management

- `sources.f` is the single source of truth for file lists and project configuration (part number, top module, output directories).
- All RTL files are in `rtl/`, all testbenches in `tb/`.
- Do not create new files in `sim/` or `viv/` — those are output directories.

## VHDL-2019: Vivado GUI Crash — Null Ranges

**NEVER use null-range signal constraints (`(-1 downto 0)`).** Vivado's interactive waveform viewer crashes with an `EXCEPTION_ACCESS_VIOLATION` when trying to enumerate zero-width signals.

Instead, use **1-bit safe-width stubs** (`(0 downto 0)`) — these are constant-driven and synthesis propagates them away (proven zero-waste).

`alias slave is master'converse` is fine (tested independently — no crash).

### Good (works):
```vhdl
signal s : axis_t(
    tdata(31 downto 0),
    tuser(0 downto 0)      -- 1-bit stub (never null)
);

view master of axis_t is ... end view;
alias slave is master'converse;             -- fine (not a crash source)
```

### Bad (crashes Vivado GUI waveform viewer):
```vhdl
signal s : axis_t(
    tdata(31 downto 0),
    tuser(-1 downto 0)     -- null range → crash
);
```

## Vivado Cruft Cleanup

Vivado and ModelSim leave journal files (`.jou`, `.log`, `transcript`, `.Xil/`) in the working directory. After any tool run:

```powershell
# Remove from current directory
del vivado*.jou, vivado*.log, transcript, vsim.wlf -EA SilentlyContinue
Remove-Item .Xil -Recurse -Force -EA SilentlyContinue
```
