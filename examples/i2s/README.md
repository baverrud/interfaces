# I2S (Inter-IC Sound) Examples

## Overview

I2S (Philips standard) is a serial bus interface for connecting digital
audio devices ‚Äî typically a CPU/DSP to an audio CODEC.  It uses three
(sometimes four) signals:

| Signal | Direction (CODEC side) | Purpose |
|--------|-----------------------|---------|
| `BCLK` | Output | Bit clock (serial shift clock) |
| `LRCLK` | Output | Left/Right word select (frame sync) |
| `tx_data` | Input | Serial audio data **to** the CODEC |
| `rx_data` | Output | Serial audio data **from** the CODEC |
| `tx_valid` | Input | Strobe: new transmit sample available |
| `rx_valid` | Output | Strobe: new receive sample captured |

In a typical system the **CODEC is the clock master** ‚Äî it generates
BCLK and LRCLK.  The CPU/DSP is the clock slave ‚Äî it receives the
clocks and synchronises its data transfers to them.

### Clock Inversion Note

The library interface (`i2s_if` / `i2s_pkg`) uses the *protocol-level*
naming convention:

| Modport / View | Role | BCLK/LRCLK direction |
|----------------|------|----------------------|
| `master` | CPU/DSP side | **Input** (receives clocks from CODEC) |
| `slave` | CODEC side | **Output** (generates clocks for CPU) |

This is **inverted** relative to what most audio engineers expect
(where the master *generates* the clocks).  To avoid confusion, the
example modules are named `controller` (CPU/DSP) and `peripheral`
(CODEC) rather than `master`/`slave`.

### Controller (`i2s_controller`)

- Connects via the **master** modport (`i2s_if.master`)
- Drives `tx_data` (24-bit audio sample `0xA5A5A5`) and `tx_valid`
- Reads `rx_data`/`rx_valid` for the echo from the peripheral
- Asserts `done` after sending 4 samples
- BCLK/LRCLK are unused inputs (generated by the peripheral)

**State machine**: `IDLE ‚Üí SEND (√ó4) ‚Üí DONE`

### Peripheral (`i2s_peripheral`)

- Connects via the **slave** modport (`i2s_if.slave`)
- **Generates BCLK** (divide system clock by 2)
- **Generates LRCLK** (divide BCLK by 64 = 32 audio frames √ó 2 channels)
- Captures `tx_data`/`tx_valid` on BCLK edges
- Echoes the received sample back as `rx_data`/`rx_valid`

### Loopback Test Flow

1. Controller drives `tx_data = 0xA5A5A5` and asserts `tx_valid`
2. Peripheral captures the sample on the next BCLK cycle
3. Peripheral drives `rx_data = 0xA5A5A5` and asserts `rx_valid`
4. Controller reads the echo (not explicitly checked ‚Äî `done` is
   asserted independently after 4 samples)

## Interface Details

### SystemVerilog (`i2s_if`)

```systemverilog
interface i2s_if #(parameter int DATA_W = 24);
  logic                bclk;
  logic                lrclk;
  logic [DATA_W-1:0]   tx_data;
  logic [DATA_W-1:0]   rx_data;
  logic                tx_valid;
  logic                rx_valid;

  modport master (       // CPU/DSP side
    input  bclk, lrclk,
    output tx_data, tx_valid,
    input  rx_data, rx_valid
  );
  modport slave (        // CODEC side
    output bclk, lrclk,
    input  tx_data, tx_valid,
    output rx_data, rx_valid
  );
endinterface
```

### VHDL-2019 (`i2s_pkg`)

```vhdl
type i2s_t is record
  bclk     : std_logic;
  lrclk    : std_logic;
  tx_data  : std_logic_vector;        -- unconstrained: width per instance
  rx_data  : std_logic_vector;        -- unconstrained: width per instance
  tx_valid : std_logic;
  rx_valid : std_logic;
end record;

view master of i2s_t is               -- CPU/DSP side
  bclk, lrclk : in;
  tx_data, tx_valid : out;
  rx_data, rx_valid : in;
end view;

alias slave is master'converse;        -- CODEC side
```

The `tx_data` and `rx_data` vectors are **unconstrained** in the record
type and must be constrained at the signal declaration:

```vhdl
signal b : i2s_t(tx_data(23 downto 0), rx_data(23 downto 0));
```

## Simulation

### Prerequisites

- Questa 2025.3 (`q26.bat`) or Vivado 2026.1 (`v26.bat`) for
  VHDL-2019 support.  ModelSim 2020.1 (`m20.bat`) can simulate the
  SystemVerilog version.

### Running

```powershell
# SystemVerilog
cd examples/i2s/sv
.\sim.bat top_tb

# VHDL-2019
cd examples/i2s/vhdl
.\sim.bat top_tb
```

### Expected Output

```
# === I2S PASSED ===
#    Time: 95 ns
# Errors: 0, Warnings: 0
```

## Testbench

The testbench (`top_tb.sv` / `top_tb.vhd`) generates a 100 MHz clock
(10 ns period), asserts reset for 4 cycles, then waits for `done` or
a 500-cycle timeout.
